# Теоретическая часть — Введение в теорию трансляторов

## 1. Что такое компилятор и зачем он нужен
Компилятор — программа, преобразующая исходный текст на языке высокого уровня в код целевой машины.
Цели: проверка корректности, оптимизация, генерация эффективного кода.

## 2. Базовые стадии
1) Лексический анализ (tokenize)
2) Синтаксический анализ (парсер → AST)
3) Семантический анализ (типы, имена, атрибуты)
4) Промежуточное представление (IR)
5) Оптимизации (IR → IR)
6) Генерация кода (Backend: выбор инструкций, регистров, расписание)
7) Ассемблер/Линковщик

## 3. LLVM и его роль
LLVM — набор библиотек/инструментов для построения компиляторов.
Ключевая идея — язык‑независимое IR (SSA), множество проходов оптимизаций, модульная архитектура.
Проект включает Clang (frontend для C/C++/ObjC), собственно LLVM (IR/оптимизатор/бэкэнды), MLIR.

## 4. AST (на примере Clang)
AST — структурное представление программы. Позволяет делать статический анализ (поиск паттернов,
диагностика), автотрансформации, рефакторинг. В Clang доступны посетители/матчеры.

## 5. LLVM IR
Типизированное SSA‑представление. Удобно для оптимизаций: инвариант петель, CSE, DCE, LICM.
Компактный текстовый формат (`.ll`) и бинарный биткод (`.bc`).

## 6. Backend
Преобразует IR в машинный код: SelectionDAG/GlobalISel, назначение регистров, выбор инструкций,
формирование MIR, затем эмиссия целевых инструкций.

## 7. MLIR
Многоуровневое IR: диалекты (arith, memref, scf, linalg, gpu, llvm). Позволяет описывать высокоуровневые
представления (Polyhedral, Tensor), а затем понижающими проходами приводить к LLVM IR.

## 8. Мини‑задачи лабораторных
- ClangAST: обойти узлы и вывести краткую статистику.
- LLVM IR: простая паттерн‑оптимизация на уровне IR.
- Backend: микрооптимизация на уровне MIR.
- MLIR: элементарная каноникализация на уровне arith.
